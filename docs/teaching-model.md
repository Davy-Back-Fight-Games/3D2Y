# Teaching Model

3D2Y uses the island as the challenge and `teach/` files as support.

The goal is not to write long tutorials. The goal is to give learners just enough knowledge to keep building.

## Teaching Loop

Each teaching file should follow this loop:

1. Explain one concept.
2. Show one tiny example.
3. Give one quick check.
4. Send the learner back to the island.

## File Location

```txt
islands/<slug>/teach/*.html
```

## Good Teaching Files

- Teach one thing only.
- Stay short enough to revisit.
- Use concrete Godot examples.
- Include a tiny practice prompt.
- Avoid solving the whole island for the learner.

## Suggested Types

- Concept lesson: explains a Godot idea.
- Reference sheet: compresses syntax or editor steps.
- Debug guide: covers common mistakes.
- Reflection guide: helps learners understand what they built.
